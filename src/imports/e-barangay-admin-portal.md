Create an Admin Portal for an E-Barangay Web System where the administrator has full control over the entire website. The system already has a Resident Portal that allows residents to submit document requests and change their password, but residents cannot manage other system data.

The Admin Portal should include the following modules:

Dashboard

Display summary statistics such as total registered residents, total document requests, pending requests, approved requests, and released documents.

Show recent activities and quick navigation to other modules.

Resident Management

Allow the admin to view, add, edit, and delete resident records.

Display resident information such as name, address, birthdate, gender, civil status, and contact number.

Include search and filter functionality.

Allow the admin to reset resident passwords if necessary.

Request Management

Display all document requests submitted by residents.

Example documents: Barangay Clearance, Certificate of Residency, Certificate of Indigency, and Business Clearance.

Allow the admin to approve, reject, process, or mark requests as released.

Include request status tracking.

Announcement Management

Allow the admin to post announcements for residents.

The admin can create, edit, delete, and publish announcements.

Announcements should be visible in the resident portal.

Reports Module

Generate reports such as resident lists, request summaries, approved documents, and released documents.

Allow reports to be viewed, printed, or exported.

Account Management

Allow the admin to change their password.

Optionally allow management of additional admin accounts.

System Settings

Allow editing of barangay information such as barangay name, logo, contact details, and website settings.

The Admin Portal should only be accessible by authorized administrators through a secure login page. Residents must not have access to admin features.
